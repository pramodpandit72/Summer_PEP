## 📋 Problem-Solving Checklist

Before submitting any solution:

- [ ] Read problem carefully (2-3 times)
- [ ] Identify the pattern/approach
- [ ] Write pseudocode first
- [ ] Code the solution
- [ ] Test with provided examples
- [ ] Test with edge cases
- [ ] Check time complexity - O(n), O(n²), O(n log n)?
- [ ] Check space complexity - O(1), O(n), O(log n)?
- [ ] Add comments for clarity
- [ ] Clean up and optimize

## 📝 Notes for Success

1. **Consistency over intensity** - 3-5 problems/day is better than 20/week
2. **Understand, don't memorize** - Focus on approach, not solution
3. **Track weak areas** - Revisit them frequently
4. **Code clean** - Good variable names, comments, organization
5. **Time yourself** - Practice under real interview conditions
6. **Teach others** - Explain your solution to someone else
7. **Optimize gradually** - Brute force → Better approach → Optimal
8. **Update regularly** - Add new problems and insights weekly
