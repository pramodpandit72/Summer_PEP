# Django Notes

## What is Django?

Django is a high-level Python web framework.

It helps developers build web applications quickly and easily.

Django follows the **"Don't Repeat Yourself (DRY)"** principle, which means writing less code by reusing existing code.

---

## Why Use Django?

- Easy to learn
- Written in Python
- Secure
- Fast development
- Built-in Admin Panel
- Supports large projects
- Handles database operations easily

---

## Prerequisites

Before learning Django, you should know:

- Basic Python
- Functions
- Classes and Objects
- Basic HTML
- Basic CSS (optional but helpful)

# Create a Django Project

Command:

```bash
django-admin startproject myproject
```

Example:

```bash

```

This creates a new Django project.

# Difference Between Project and App

## Project

A complete website.

Example:

```
College Website
```

---

## App

A single feature of a website.

Examples:

- Blog
- Login
- Student Management
- Attendance
- Payment

One project can contain many apps.

---

# App Folder Structure

```
blog/
│
├── admin.py
├── apps.py
├── models.py
├── tests.py
├── views.py
├── migrations/
└── __init__.py
```

---

## views.py

Contains business logic.

Example:

```python
from django.http import HttpResponse

def home(request):
    return HttpResponse("Hello Django")
```

---

## models.py

Defines database tables.

Example:

```python
from django.db import models

class Student(models.Model):
    name = models.CharField(max_length=100)
```

---

## admin.py

Registers models so they appear in the Django Admin Panel.

Example:

```python
from django.contrib import admin
from .models import Student

admin.site.register(Student)
```

---

## apps.py

Stores app configuration.

Usually, you do not edit this file.

---

## migrations Folder

Stores database migration files.

Do not delete migration files unless you know what you are doing.

---

# Templates

Templates contain HTML files.

Create:

```
templates/
```

Example:

```
templates/
    home.html
```

View:

```python
from django.shortcuts import render

def home(request):
    return render(request, "home.html")
```

# Models

Models represent database tables.

Example:

```python
class Student(models.Model):
    name = models.CharField(max_length=100)
    age = models.IntegerField()
```

Each object represents one row in the database.

# Important Interview Questions

### 1. What is Django?

Django is a Python web framework used to build web applications quickly.

---

### 2. What is a Project?

A project is the complete website.

---

### 3. What is an App?

An app is a single module or feature inside a project.

---

### 4. What is a Model?

A model represents a database table.

---

### 5. What is a View?

A view contains the business logic and returns a response.

---

### 6. What is a Template?

A template is an HTML file shown to the user.

---

### 7. What is URL Routing?

URL routing connects a URL with a view.

---

Migrations update the database according to model changes.

---

### 9. What is the Django Admin Panel?

A built-in dashboard to manage database records.

---

### 10. What is manage.py?

A command-line utility used to perform Django tasks.

# Summary

- Django is a Python web framework.
- A project is a complete website.
- An app is one feature of the project.
- Models work with the database.
- Views contain application logic.
- Templates display HTML pages.
- URLs connect browser requests to views.
- Migrations create and update database tables.
- The Admin Panel helps manage data.
